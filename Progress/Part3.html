Outliers can skew model predictions results, so I remove any rows with extremely low sale prices. There are multiple rows with sale price values less than 100, which doesn’t make sense because the units are in dollars and most sale prices are at least $100,000.

The last step in pre-processing the dataset is normalizing the data. This step isn’t always required, but since many columns’ magnitudes dominates others, models can’t pick up the contribution of the smaller magnitude features. Thus, in my case, I have to standardize the features. I used the preProcess() function from the caret package to first calculate the pre-process parameters then predict() to transform the dataset using those parameters.

Scaling

manhattan_final <- predict(preProcess(manhattan_final, method=c("range")), manhattan_final)

I can now go on to partition the dataset into two sets: a training set (70% of the dataset) and a testing set (30%).

Partition data

set.seed(600)
in_training <- createDataPartition(manhattan_final$SALE.PRICE, p = 0.70, list=FALSE)
training <- droplevels(manhattan_final[in_training,])
testing <- droplevels(manhattan_final[-in_training,])

For each model, I calculate the mean absolute error (MAE) so that I can pick the model with the smallest MAE. I will be fitting the following models on the data:

Regression Models:

  1. OLS

  2. Poisson

Tree-Based Models:

  1. Random Forest

  2. GBM

OLS

fit_ols <- lm(SALE.PRICE ~ ., data = training)

y_hat_ols <- predict(fit_ols, testing)
ols_mae <- mae(testing$SALE.PRICE, y_hat_ols)

Poisson

fit_poisson <- glm(SALE.PRICE ~ ., data = training, family="poisson")

y_hat_poisson <- predict(fit_poisson, testing, type = "response")
pois_mae <- mae(testing$SALE.PRICE, y_hat_poisson)

Random Forest

fit_rf <- randomForest(SALE.PRICE ~ ., data = training)

y_hat_rf <- predict(fit_rf, testing)
rf_mae <- mae(testing$SALE.PRICE, y_hat_rf)

While random forests build an ensemble of deep independent trees, GBM (generalized boosted machine) models build one of shallow and weak successive trees, each tree learning and improving on the previous. The below gbm model generates 10000 trees, has a shrinkage parameter (learning rate) of 0.01, and has an interaction depth–the total splits–to 4, so each tree is a small tree with only 4 splits. Furthermore, I chose a gaussian distribution to minimize squared error.

GBM

fit_gbm <- gbm(SALE.PRICE ~ ., data = training ,distribution = "gaussian", n.trees = 10000, shrinkage = 0.01, interaction.depth = 4)

y_hat_gbm <- predict(fit_gbm, testing, n.trees = 10000)
gbm_mae <- mae(testing$SALE.PRICE, y_hat_gbm)

Here are the final results from all the models:

results <- data.frame()
ols <- c("OLS", ols_mae)
pois <- c("Poisson", pois_mae)
rf <- c("Random Forest", rf_mae)
gbm <- c("Gradient Boosting", gbm_mae)

results <- rbind(ols, pois, rf, gbm)
rownames(results) <- c()
colnames(results) <- c("Model", "Mean Absolute Error (MAE)")
